# Definition of Done

Una tarea se considera terminada solo cuando:

1. Cumple todos los criterios de aceptación.
2. No rompe funcionalidad existente.
3. Tiene pruebas relevantes.
4. Pasa lint/análisis estático.
5. Pasa CI.
6. Fue revisada.
7. No introduce deuda técnica innecesaria.
8. Actualiza documentación si cambia comportamiento, arquitectura, datos o seguridad.
9. Registra riesgos pendientes.
10. El usuario puede validar el resultado.

Para tareas relacionadas con IA, además:

11. La salida está validada por schema.
12. Las suposiciones se marcan.
13. Las decisiones críticas requieren aprobación humana.
14. Se registra agent_run.
