# Definition of Ready

Una tarea está lista para desarrollo cuando cumple:

1. Objetivo claro.
2. Contexto suficiente.
3. Criterios de aceptación escritos.
4. Dependencias identificadas.
5. Diseño aprobado si afecta arquitectura.
6. Modelo de datos definido si persiste información.
7. Seguridad considerada.
8. Pruebas requeridas identificadas.
9. No hay contradicciones críticas abiertas.
10. El alcance de escritura está definido.

Si falta alguno de estos puntos, la tarea debe volver a refinamiento.
