# MVP Scope — Technical Foundation Builder

## 1. Objetivo del MVP

Validar si la plataforma puede transformar una idea informal en un paquete técnico inicial suficientemente útil para comenzar la planificación de desarrollo.

## 2. Incluido en el MVP

### Cuenta y acceso

- Registro.
- Login.
- Logout.
- Sesión persistente.

### Organización

- Crear organización.
- Asociar proyectos a organización.
- Rol owner inicial.

### Proyecto

- Crear proyecto.
- Editar datos básicos.
- Ver dashboard del proyecto.

### Entrevista

- Descripción inicial de la idea.
- Preguntas generadas por IA.
- Respuestas guardadas.
- Progreso por etapa.
- Preguntas pendientes.

### Extracción estructurada

- Requisitos.
- Reglas de negocio.
- Entidades.
- Supuestos.
- Riesgos.
- Decisiones.
- Criterios de aceptación.

### Generación documental

Documentos iniciales:

1. PRODUCT_SPEC.md
2. MVP_SCOPE.md
3. DOMAIN_MODEL.md
4. ARCHITECTURE.md
5. DATA_MODEL.md
6. SECURITY.md
7. BACKLOG.md
8. VERTICAL_SLICE_PLAN.md

### Validación

- Detección básica de información faltante.
- Detección básica de contradicciones.
- Readiness Score inicial.

### Exportación

- Markdown.
- PDF.
- ZIP.

## 3. No incluido en el MVP

- Cobros o suscripciones.
- Múltiples miembros por organización.
- Comentarios colaborativos.
- Integración GitHub.
- Integración Jira/Linear.
- Generación de código.
- Plantillas comerciales avanzadas.
- Marketplace de expertos.
- Soporte multiidioma completo.
- App móvil.
- Modo offline.
- Compliance regulatorio avanzado.
- Edición visual avanzada de diagramas.

## 4. Usuarios iniciales para validación

Prioridad:

1. Agencias pequeñas de desarrollo.
2. Consultores técnicos.
3. Fundadores no técnicos con proyectos reales.
4. Freelancers full stack.
5. Desarrolladores junior/intermedios.

## 5. Métricas de validación

- Tiempo para pasar de idea a paquete exportado.
- Porcentaje de usuarios que completan la entrevista.
- Cantidad de preguntas pendientes detectadas.
- Cantidad de supuestos marcados.
- Satisfacción del usuario con los documentos generados.
- Utilidad percibida por desarrolladores que reciben el paquete.
- Intención de pago o reutilización.

## 6. Primer vertical slice

Usuario se registra → crea organización → crea proyecto → describe idea → responde preguntas → sistema extrae requisitos → genera PRODUCT_SPEC.md → calcula readiness parcial → exporta Markdown.
